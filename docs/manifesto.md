# Project Manifesto — Sasa/Zamani

We're building a meaning-making tool grounded in John Mbiti's philosophy of Bantu time. The tool takes raw lived experience — Telegram messages, conversation transcripts — and, using AI-assisted semantic clustering and Campbell's narrative archetypes as scaffold, helps individuals and communities transform that data into myth: compressed, portable, functionally true narrative that tells you what your life is mythologically about.

The system has four layers: raw data (sasa), archetype pattern recognition, falsifiable truth candidates, and myth output. The core visualization — the Sasa Map — renders time as a pool organized by semantic resonance rather than chronology, where events cluster into constellations that generate mythic language. A working HTML prototype exists with two views (strata and resonance), archetype glyphs, Claude API integration for myth generation, and a tag-based similarity engine ready to be replaced with real embeddings.

The team: Jessie (design lead, original concept), Emma (branding, copy, thematic guidance), Steven (technical lead, development, philosophical orientation). The initial context is Edge City popup village gatherings, with individual use as the primary long-term use case.

**Constraints:** Three-person team. Steven is sole developer. No external funding. MVP target: one week (for Edge City demo). The product must feel like looking into water, not looking at a dashboard.

**What we know:** Philosophical framework is solid and well-articulated. Visual identity and design language are defined. A working prototype exists. The four-layer model is agreed upon. Campbell archetypes are the starting scaffold with room for novel archetypes to emerge.

**What we don't know:** Whether the embedding model produces meaningful clusters on real natural-language input. What the minimum viable habit loop is. How truth candidates (Layer 3) and full myth output (Layer 4) work in implementation. The name. What a five-year mythological map looks like.
