# Dev Log — Sprint 3.5, Fix Session F1: Segmentation max_tokens and client timeout

**Date:** 2026-03-19

## What Changed

Fixed two issues in `app/segmentation.py` that blocked S3 (re-seed):

1. **max_tokens too low:** `segment_transcript()` used `max_tokens=4096`, which
   truncated Claude's JSON response for large transcripts (March 17 ~80K chars,
   March 18 ~136K chars). Changed to `max_tokens=32000`.

2. **No explicit timeout:** The Anthropic client had no timeout, causing the
   SDK's auto-calculated timeout to behave unpredictably at high max_tokens
   values. Added `timeout=120.0` to the client constructor.

## Files Modified

- `app/segmentation.py` — max_tokens 4096→32000 in segment_transcript(); timeout=120.0 in _create_client()
- `tests/test_segmentation.py` — added test_segment_transcript_max_tokens regression test
