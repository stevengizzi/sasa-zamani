# Sprint 4, Session 1: Segmentation Core

**Date:** 2026-03-19
**Sprint:** 4 — Data Quality + Significance Filtering
**Session:** 1 — Prompts, Dataclass, Significance, Labels, Dedup

## What Changed

1. **Redesigned segmentation and label prompts** — Labels now follow marginalia
   register (4-8 word propositions, not tag clouds). Both prompts include
   significance scoring (0.0–1.0) and the segmentation prompt enforces label
   uniqueness.

2. **Extended Segment dataclass** — Added `start_line`, `end_line`, and
   `significance` fields with backward-compatible defaults.

3. **Updated `segment_transcript()`** — Parses significance from Claude response,
   defaults to 1.0 when missing, clamps out-of-range values with warnings.
   Populates start_line/end_line on each Segment.

4. **Added `filter_by_significance()`** — Pure function, returns segments at or
   above a threshold without mutating input.

5. **Added `dedup_labels()`** — Appends Roman numeral suffixes to duplicate labels
   without mutating input.

6. **Updated `generate_event_label()`** — Now returns `tuple[str, float]` instead
   of `str`. Parses JSON response with label + significance. max_tokens bumped
   from 50 to 100.

7. **Added `significance_threshold` to config** — Defaults to 0.3.
