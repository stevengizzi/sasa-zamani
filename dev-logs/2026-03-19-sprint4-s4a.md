# Sprint 4, Session 4a: Granola Pipeline Integration

**Date:** 2026-03-19
**Sprint:** 4 — Data Quality + Significance Filtering
**Session:** 4a — Wire new pipeline steps into Granola upload

## What Changed

1. **Raw input storage** — `process_granola_upload` now calls `insert_raw_input`
   before segmentation, storing the full transcript with source metadata. The
   returned `raw_input_id` is passed through to `insert_event`.

2. **Label dedup** — After segmentation, `dedup_labels` is called to append
   ordinal suffixes to duplicate labels within a transcript.

3. **Significance filtering** — After dedup, `filter_by_significance` removes
   segments below the configured threshold (default 0.3).

4. **Dynamic cluster creation** — `assign_cluster` replaced by
   `assign_or_create_cluster`. When a new cluster is created, centroids are
   re-fetched so subsequent segments in the same batch can join it.

5. **Archetype naming** — `maybe_name_cluster` is called after
   `increment_event_count`. Failures are logged but do not block the pipeline.

6. **Event lineage** — `insert_event` now receives `raw_input_id`,
   `start_line`, and `end_line` for traceability back to the raw transcript.

## Pipeline Order

store raw → segment → dedup labels → filter significance → embed → assign/create → insert → increment → name → xs
