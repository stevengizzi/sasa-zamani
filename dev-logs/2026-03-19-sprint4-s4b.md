# Sprint 4, Session 4b: Seed Script Integration

**Date:** 2026-03-19
**Sprint:** 4 — Data Quality + Significance Filtering
**Session:** 4b — Wire Sprint 4 features into batch seeding script

## What Changed

1. **Significance filtering replaces min-length** — Removed `--min-length` CLI arg
   and `filter_by_length()` function. Segments are now filtered by their LLM-assigned
   significance score against `significance_threshold` (default 0.3).

2. **Label deduplication** — `dedup_labels()` applied after segmentation, before
   significance filtering, matching the Session 4a granola.py pipeline pattern.

3. **Raw input storage** — Transcript text stored in `raw_inputs` table via
   `insert_raw_input()` before pipeline processing. Returned `raw_input_id` passed
   through to `insert_event()` along with `start_line`/`end_line`.

4. **Dynamic cluster creation** — `assign_or_create_cluster()` replaces
   `assign_cluster()`. When a new cluster is created, centroids are refreshed
   mid-batch so subsequent segments can join the new cluster.

5. **Deferred naming** — `maybe_name_cluster()` called after each
   `increment_event_count()`, wrapped in try/except with warning log on failure.

6. **Enhanced dry-run output** — Shows significance scores for all segments,
   marks below-threshold segments with `[FILTERED]`, and displays summary count.
