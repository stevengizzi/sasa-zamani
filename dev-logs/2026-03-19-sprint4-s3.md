# Sprint 4, Session 3: Archetype Creation Core

**Date:** 2026-03-19
**Sprint:** 4 — Data Quality + Significance Filtering
**Session:** 3 — Below-threshold cluster creation + deferred archetype naming

## What Changed

1. **Dynamic cluster creation** — When an embedding's best cosine similarity is
   below `cluster_join_threshold`, `assign_or_create_cluster` now creates a new
   cluster with the embedding as centroid, `is_seed=False`, and placeholder name
   "The Unnamed".

2. **Deferred archetype naming** — New `app/archetype_naming.py` module. When a
   cluster reaches `archetype_naming_threshold` events (default 3) and is still
   named "The Unnamed", `maybe_name_cluster` calls Claude to generate an
   archetype name in the ancestral register.

3. **Config** — Added `archetype_naming_threshold: int = 3` to Settings.

4. **DB functions** — Added `update_cluster_name` and `get_cluster_events_notes`
   (truncates notes to 200 chars each).

## Decisions

- `assign_or_create_cluster` replicates the similarity loop from `assign_cluster`
  rather than calling it, keeping both functions decoupled.
- `create_dynamic_cluster` uses a local import of `insert_cluster` to avoid
  circular imports (same pattern as `seed_clusters`).
