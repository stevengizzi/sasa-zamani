# Dev Log — Sprint 3.5, Fix Session F2: Boundary-Based Segmentation Prompt

**Date:** 2026-03-19

## What Changed

Redesigned the segmentation prompt internals to eliminate verbatim text output:

1. **Line-numbered input:** Transcript is now sent with `L001:` prefixes so Claude
   can reference line numbers instead of reproducing text.

2. **Boundary-based output:** Claude returns `{start_line, end_line, label, speakers}`
   instead of `{text, label, speakers}`. Python slices original lines to reconstruct
   segment text.

3. **Boundary validation:** Added checks for start > end, out-of-range lines, and
   overlapping segments — all raise `SegmentationError`.

4. **max_tokens reduced:** 32000 → 4096. With boundary output, even the largest
   transcript produces < 1000 output tokens.

This resolves the root cause of the F1 issue: Claude was generating ~25K output
tokens to echo back the entire transcript, causing timeouts and high cost.

- `app/segmentation.py` — new prompt, boundary parsing, validation, max_tokens 32000→4096
- `tests/test_segmentation.py` — 5 existing mocks updated, 4 new boundary tests added
