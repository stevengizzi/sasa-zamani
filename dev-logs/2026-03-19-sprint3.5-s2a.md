# Dev Log: Sprint 3.5, Session S2a — Granola + Seed Transcript Pipeline Integration

**Date:** 2026-03-19
**Sprint:** 3.5
**Session:** S2a

## What Changed

Replaced regex-based speaker-turn parsing in both `app/granola.py` and
`scripts/seed_transcript.py` with thematic segmentation from `app/segmentation.py`.
Events now get LLM-generated labels and multi-participant attribution.

### Changes
- `app/granola.py`: Removed `parse_transcript()` and regex imports. `process_granola_upload()`
  now calls `segment_transcript()`, derives participant from segment.participants list,
  uses segment.label for event labels, passes participants array to `insert_event()`.
- `scripts/seed_transcript.py`: Same replacement. `filter_by_length()` updated to work
  with `Segment` dataclass. Dry-run prints thematic labels and speaker attribution.
- `tests/test_granola.py`: Rewrote to mock `segment_transcript`. 7 new tests covering
  segmentation integration, multi/single speaker logic, error propagation, label source.
- `tests/test_seed_transcript.py`: Rewrote to mock `segment_transcript`. 2 new tests
  for segmentation integration and dry-run behavior.

## Test Results
- Scoped: 21 passed (tests/test_granola.py + tests/test_seed_transcript.py)
- Full suite: 156 passed, 3 skipped, 3 pre-existing errors

## Notes
- Resolves DEF-019 (LLM-generated event labels instead of raw text[:80]).
